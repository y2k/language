## Context

`Macro_ns.apply` распознаёт `:import` только при точной форме с одним bracket-вектором. Макрос уже накапливает import pairs и сохраняет их порядок перед созданием `compiler/ns`; Java-бэкенд преобразует каждый pair в обычный Java `import`.

## Goals / Non-Goals

**Goals:**

- Принимать один или несколько bracket-векторов после `:import`.
- Сохранять import pairs в порядке векторов и классов в исходном тексте.
- Не менять результат и ошибки существующей одно-векторной формы.
- Проверять весь Java-путь через deterministic sample, который `javac` компилирует и выполняет.

**Non-Goals:**

- Не добавлять aliases, wildcard- или static-import syntax.
- Не менять обработку import pairs в Java, JavaScript или eval-бэкендах.
- Не вводить таблицу импортированных типов или разрешение символов во frontend.

## Decisions

### Разобрать список import-векторов в существующем макросе

`Macro_ns.apply` будет сопоставлять `:import` с хвостом из одного или нескольких bracket-векторов. Для каждого вектора первый символ остаётся package, последующие символы становятся именами классов; для каждого класса создаётся прежний pair `(Class "package.Class")`.

Это минимально расширяет грамматику и сохраняет договор между frontend и Java-бэкендом. Альтернатива с новым внутренним представлением групп импорта не нужна: Java уже потребляет плоский список pairs.

### Сохранить текущую валидацию формы

Каждый элемент после `:import` должен быть bracket-вектором, а имена классов в нём должны быть symbols. Неверные формы продолжают завершаться `ns_error` с сообщением о допустимых `:require` и `:import` clauses.

Это не расширяет язык неоговорёнными формами и не меняет правила для существующего `(:import [package Class ...])`.

### Проверять дешугорирование и скомпилированный Java-код

Unit-тест зафиксирует flat import pairs для двух векторов в одном clause. Java sample задействует классы из разных импортированных пакетов и пройдёт существующий `javac`/`java` harness.

Text-only assertion Java-генерации недостаточна: она не доказывает, что short class names успешно разрешаются компилятором Java.

## Risks / Trade-offs

- [Потеря порядка импортов при накоплении списка] → Проверить точный порядок pairs в frontend unit-тесте.
- [Тест использует недетерминированный Java API] → Выбрать статические вызовы с фиксированным результатом.
- [Ожидание aliases или wildcard imports] → Явно оставить их вне scope; они потребуют отдельного синтаксического решения.
